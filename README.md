# AI Hallucination Detector

A production-ready tool to detect potential hallucinations in AI-generated text using multiple detection methods.

## Features

- **Perplexity Analysis** - Measures how unusual/unnatural the text sounds using GPT-2
- **AI Text Detection** - Detects AI-generated text using RoBERTa (OpenAI detector)
- **Self-Consistency** - Verifies internal consistency by generating multiple responses (Groq API)
- **Fact Verification** - Cross-references facts with Wikipedia API

## Tech Stack

- **Perplexity**: GPT-2 (HuggingFace, free, CPU)
- **AI Text Detection**: roberta-base-openai-detector (HuggingFace, free, CPU)
- **LLM**: Groq API (Llama-3.1-8B-Instant, free tier)
- **Fact Check**: Wikipedia API (free)
- **Frontend**: Streamlit
- **Backend**: FastAPI ready

## How It Works

1. **Input**: Paste any text to analyze
2. **Perplexity**: GPT-2 scores how "natural" each sentence sounds
3. **AI Detection**: RoBERTa detects if text is AI-generated
4. **Consistency**: Multiple LLM generations are compared for contradictions
5. **Fact Check**: Claims are verified against Wikipedia
6. **Output**: Overall hallucination score + flagged sentences

## How to Analyze Output

### 1. Overall Hallucination Score (0% - 100%)
- **0-30% (Low Risk):** Text appears mostly reliable. Normal AI-generated content.
- **30-50% (Medium Risk):** Some questionable claims detected. Review flagged sentences.
- **50-100% (High Risk):** Strong indicators of hallucination. Do NOT use without verification.

### 2. Perplexity Analysis (GPT-2)
Measures how "natural" each sentence sounds compared to GPT-2's training data.

| Score | Interpretation |
|-------|---------------|
| < 50 | Very natural, likely human-written |
| 50 - 100 | Normal AI-generated text |
| 100 - 150 | Slightly unusual phrasing |
| > 150 | Highly unusual — possible hallucination |

**Tip:** High perplexity = the sentence has unusual word patterns, which AI sometimes produces when fabricating information.

### 3. AI Text Detection (RoBERTa)
Detects if text was likely generated by an AI model.

| Label | Interpretation |
|-------|---------------|
| Real | Likely human-written |
| Fake | Likely AI-generated |

**Tip:** 100% AI-generated text isn't necessarily wrong — it's just AI-generated. Combine with other metrics.

### 4. Self-Consistency Score (Groq API)
Checks if the same claims get consistent answers across multiple LLM generations.

| Score | Interpretation |
|-------|---------------|
| > 0.7 | High internal consistency — claims are stable |
| 0.4 - 0.7 | Some inconsistencies — verify key facts |
| < 0.4 | Low consistency — possible fabrication |

**Tip:** Add your Groq API key in the sidebar for self-consistency checks.

### 5. Fact Verification (Wikipedia)
Cross-references factual claims against Wikipedia.

| Rate | Interpretation |
|------|---------------|
| > 70% | Most facts verified |
| 40-70% | Some facts unverifiable |
| < 40% | Most facts not verified — HIGH RISK |

**Tip:** Wikipedia may miss niche topics. Unverified ≠ false.

### 6. Flagged Sentences
Sentences that triggered at least one high/medium risk indicator. Each flag shows:
- **Risk Level:** How concerning the issue is
- **Reason:** What caused the flag (perplexity, AI-detection, fact-check, etc.)

### Recommended Workflow
1. Check **Overall Score** first — is it Low/Medium/High?
2. Read **Flagged Sentences** — what exactly triggered the risk?
3. Expand **Detailed Metrics** — which detection method flagged it?
4. For High Risk text:
   - Do NOT use without manual verification
   - Cross-reference claims with authoritative sources
   - Consider rewriting flagged sections
